# Mini NPU Simulator

## 프로젝트 개요
뇌의 신경세포인 '뉴런'은 들어온 모든 입력값에 각각의 가중치를 곱하고, 그 결과들을 전부 더합니다. 이후 약간의 보정 값(편향, bias)을 더해줍니다.
<br>
뉴런 결과 = (x1*w1) + (x2*w2) + (x3*w3) + ... + (bias)
<br>
<br>
그렇게 계산된 값이 활성화 함수(임계값)을 통과하여 최종 출력값이 되고, 이를 입력값으로 다시 연산하는 것을 반복해 최종 결론을 내립니다.
<br>
ex) 1계층[점, 선] -> 2계층[눈, 코, 귀] -> 3계층[얼굴 전체] -> 4계층[고양이] -> 이 조합은 '고양이'일 확률이 98%다 라는 결론
<br>
<br>
이러한 신경망 모델을 본 뜬 AI 딥러닝 모델에서도, 일어나는 연산의 대다수는 MAC 연산(곱하고 누적하는 연산)의 반복입니다.
<br>
기존 CPU는 복잡하고 논리적인 순차적 업무를 잘 처리하지만, 수천 수만개의 단순 계산을 한꺼번에 처리하는 데는 비효율적입니다.
<br>
반면 NPU는 수많은 단순 계산을 병렬로 처리하는 것에 특화되어 만들어졌습니다.
<br>
<br>
이 프로젝트에서는 MAC 연산의 핵심 원리를 구현하며 이해하고, 데이터 크기에 따른 연산 시간의 변화에 대해 알아봅니다.

<br>

### 구현 요약
1. 라벨 정규화
<br>

[개요]

<br>

사용자나 데이터 소스에 따라 동일한 의미의 라벨이 'cross', '+', 'Cross'처럼 다양한 형태로 표현될 수 있습니다. 
<br>
이렇게 일관성 없는 데이터를 그대로 사용하면 모델이 정확하게 패턴을 학습하고 평가하기 어렵습니다.
<br>
라벨 정규화는 이렇게 제각각인 라벨들을 하나의 표준화된 형태로 통일시켜주는 필수적인 전처리 과정입니다.

<br>
<br>

[코드]
```
# 표준 라벨로 정규화
def normalize_label(expected_raw):
    # 1. 문자열로 변환 후, 양 끝 공백 제거 및 소문자 통일
    label = str(expected_raw).strip().lower()
    
    # 2. '십자가' 패턴에 해당하는 라벨들을 "Cross"로 통일
    if label in ['cross', '+', 'cross']: # 'Cross'는 소문자화되어 'cross'가 됨
        return "Cross"
    # 3. 'X' 패턴에 해당하는 라벨들을 "X"로 통일
    elif label in ['x']: # 'X'는 소문자화되어 'x'가 됨
        return "X"
    
    # 4. 위 규칙에 해당하지 않으면 "UNDECIDED"로 처리
    return "UNDECIDED"
```

<br>
<br>

[설명]
- 표준화: strip()과 lower() 함수를 이용해 입력된 라벨의 양 끝 공백을 없애고 모두 소문자로 바꿔서, 대소문자나 공백으로 인한 오류를 원천 차단합니다.
- 매핑: if/elif 문을 통해 다양한 형태의 라벨들을 미리 정해둔 대표 라벨("Cross", "X")로 매핑합니다.
- 예외 처리: 정의된 규칙에 맞지 않는 라벨이 들어올 경우, "UNDECIDED"로 처리하여 프로그램의 안정성을 높입니다.

2. MAC 연산 구현
<br>

[개요]

<br>

입력 데이터(패턴)의 각 요소와 필터(가중치)의 각 요소를 서로 곱한 뒤, 그 결과들을 모두 더하여 최종 점수(Score)를 계산합니다.
<br>
이 점수는 입력 패턴이 필터가 찾고자 하는 특징과 얼마나 유사한지를 나타내는 척도가 됩니다.

<br>
<br>

[코드]
```
# 패턴과 필터의 MAC(곱셈-누적 연산)
def calculate_mac(pattern, filter_matrix):
    score = 0.0
    # 2중 for 루프를 통해 2차원 배열의 모든 요소를 순회
    for i in range(len(pattern)):
        for j in range(len(pattern[0])):
            # 같은 위치의 값을 곱하고(Multiply) score에 누적(Accumulate)
            score += pattern[i][j] * filter_matrix[i][j]
    return score
```

<br>
<br>

[설명]
- 2중 for 루프를 사용하여 2차원 행렬인 pattern과 filter_matrix의 모든 (i, j) 위치를 하나씩 방문합니다.
- 각 위치에서 pattern[i][j] 값과 filter_matrix[i][j] 값을 곱하고(Multiply), 그 결과를 score 변수에 계속해서 더해 나갑니다(Accumulate).
- 모든 위치의 계산이 끝나면, 최종적으로 누적된 score 값이 반환됩니다.

3. 동점 처리 정책
   
<br>

[개요]

<br>
컴퓨터에서 소수점(부동소수점)을 연산할 때는 아주 미세한 오차가 발생할 수 있습니다.
<br>
10진법 소수를 컴퓨터가 사용하는 2진법으로 완벽하게 바꾸지 못하는 경우가 많기 때문입니다.
<br>
ex) 0.1 (10진수) = 0.0001100110011... (2진수)
<br>
컴퓨터 메모리는 유한하기 때문에 이 무한한 숫자를 다 저장할 수 없어, 정해진 길이에서 숫자를 잘라내고 가장 가까운 값으로 저장합니다.
<br>
따라서 두 점수가 완벽하게 같은지 score_a == score_b로 비교하면, 미세한 오차 때문에 같다고 판단해야 할 상황을 놓칠 수 있습니다.
<br>
이를 방지하기 위해 **엡실론(Epsilon)**이라는 아주 작은 허용 오차 값을 두고, 두 값의 차이가 이 엡실론보다 작으면 사실상 '동점'으로 간주하는 정책을 사용합니다.

<br>
<br>

[코드]
```
# 표준 라벨로 정규화
# 허용 오차 설정 (1e-9: 0.000000001)
EPSILON = 1e-9

# mac 점수 비교 후 판정 결과 반환
def compare_scores(score_a, score_b):
    # 두 점수 차이의 절댓값이 EPSILON보다 작으면 동점으로 처리
    if abs(score_a - score_b) < EPSILON:
        return "UNDECIDED"
    # 동점이 아닐 경우, 더 큰 점수를 가진 쪽을 판정
    elif score_a > score_b:
        return "A"
    else:
        return "B"
```

<br>
<br>

[설명]
- 동점 판정: abs(score_a - score_b)를 통해 두 점수 차이의 절댓값을 구합니다. 이 차이가 설정한 EPSILON 값보다 작으면, 연산 오차 범위 내에 있는 것으로 보고 "UNDECIDED"(결정 불가)로 처리합니다.
- 승패 판정: 동점이 아닐 경우에는 두 점수를 비교하여 더 높은 점수를 받은 쪽의 결과를 반환합니다.

<br>
<br>

3. 성능 및 시간 복잡도 분석
- 입력 데이터(패턴)의 크기가 커질수록 연산 횟수와 평균 실행 시간이 함께 증가하는 명확한 경향을 보여줍니다.
- 시간복잡도: 이 프로그램의 핵심 연산인 calculate_mac 함수의 시간 복잡도는 O(N²) 입니다. calculate_mac 함수는 N x N 크기의 2차원 배열을 처리하기 위해 2중 for 루프를 사용합니다. 바깥쪽 루프가 N번, 안쪽 루프가 N번 반복되므로 총 연산 횟수는 N * N = N²에 비례합니다. 성능 측정 결과를 통해서도 명확히 확인할 수 있습니다.
